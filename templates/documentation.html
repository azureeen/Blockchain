Blockchain Application Documentation

Overview

This document provides an overview of the Flask-based blockchain application. It covers the main components and functionalities such as block creation, transaction handling, and the proof of work algorithm.

Blockchain Class

The Blockchain class is the backbone of the application. It maintains the ledger, handles new transactions, and creates new blocks.

Key Methods:

  • new_block: Creates and adds a new block to the chain.
  • new_transaction: Adds a new transaction to the current transactions list.
  • proof_of_work: Implements the Proof of Work algorithm.
  • valid_proof: Validates a proof given the last proof.
  • show_last_10_transactions: Returns the last 10 transactions.
  • tamper: Alters a transaction for demonstration purposes.
  • check_transaction: Retrieves a specific transaction by its ID.

Genesis Block

The genesis block is the first block in the blockchain. It is created with a predefined proof and a previous hash.

Proof of Work Algorithm

The proof of work algorithm requires finding a number that, when hashed with the previous block's proof, produces a hash with four leading zeroes.

Flask Routes

The application exposes several HTTP endpoints to interact with the blockchain.

Main Endpoints:

  • /transactions/new: Endpoint to submit a new transaction.
  • /chain: Endpoint to get the full blockchain.
  • /transactions/last: Endpoint to get the last 10 transactions.

Detailed documentation of each API endpoint, including request/response formats and example calls.

Architecture Overview

This section provides a high-level overview of the blockchain architecture, focusing on the key components that make up the blockchain network.

Key Components:

  • Nodes: Each participant in the network operates a node. Nodes are responsible for maintaining a copy of the blockchain and participating in blockchain operations like transaction validation and block creation.
  • Blockchain: A decentralized and distributed ledger that records all transactions across the network. The blockchain is composed of a series of blocks linked together in a chain.
  • Blocks: Each block contains a set of transactions, a reference to the previous block (previous hash), and a unique solution to a cryptographic challenge (proof of work).
  • Transactions: The primary data structure that represents value transfer between participants. Transactions are validated and then added to a block.
  • Consensus Algorithm: A mechanism that ensures all nodes agree on the blockchain's current state. This Flask blockchain application uses a Proof of Work algorithm.

Transaction Management

This section explains how transactions are created, validated, and added to the blockchain.

Creating Transactions:

Transactions are initiated by network participants. Each transaction includes essential information:

  • Sender: The address or identifier of the party sending value.
  • Receiver: The address or identifier of the receiving party.
  • Amount: The value or quantity being transferred.

Transactions are broadcast to the network where nodes validate them based on predefined criteria such as signature validity and sufficient balance.

Adding to Blocks:

Once validated, transactions are grouped together into a block. The process of creating a new block involves solving a cryptographic challenge (proof of work). The first node to solve this challenge broadcasts the new block to the network, where other nodes verify and add it to their copy of the blockchain.

This process ensures that all transactions are recorded in a decentralized and tamper-resistant manner, providing security and integrity to the blockchain network.

Block Structure

Detailed description of the block data structure in the blockchain.

  • Index: The position of the block in the chain.
  • Timestamp: The time at which the block was created.
  • Transactions: A list of transactions included in the block.
  • Proof: The proof of work associated with the block.
  • Previous Hash: The hash of the previous block in the chain.
